ü§å Why PACE?

Podcasts usually contains a lot of information, but it's hard to find what you're looking for. PACE is a semantic search engine with AI superpowers that helps you find the information you need, in a fast and easy way.

To get more superpowers, enable AI-based chapters and let PACE automatically search for the most important parts of the podcast. Even more, you can enjoy the magic of AI by letting PACE automatically generate chapter artworks for each podcast episode.

It is a project created for the AssemblyAI 50K Hackathon - Winter 2022.


üîé How does it work?

Podcasts usually are long-form audio content. They could contain many interesting topics, but you may be interested in only one or some of them.

PACE got you covered!

  1. üîà Speech Processing: generate chapters from audio tracks
  2. üìÉ NLP: search for episodes or chapters using natural language
  3. üé® Computer Vision: generate artwork for chapters using text-to-image models
  4. üíª Web: Flask + Elasticsearch to build a web app


üö® Disclaimer

Given the limited time of the hackathon, we had to use limited resources for the project.

  • üì∞ News-related podcasts only: we crawled episodes from more than 23K podcast series. However, it was simply not feasible to index all these episodes in the time we had. We focused on news-related episodes (~1500 podcast series reaching a total of ~350K episodes).
  • ‚ú® Auto-chapters only for a few podcast series: Chapter detection is based on amazing APIs offered by AssemblyAI. However, we had to limit the number of requests we could make to the API. Therefore, we focus on 2 podcast series (The Daily Good and Daily News Brief). The total number of episodes with chapter-level semantic information is ~300 (~2500 chapters).
  • ‚ö´ Cohere-backed chapter search: superpowers are powered by Cohere and AssemblyAI. Chapter are captioned using AssemblyAI and embedded using the Cohere's Language Models. For episodes semantic search, PACE uses sentence-transformers and state-of-the-art open source models.
  • üñºÔ∏è Stable Diffusion artwork generation: text-to-image AI models have reached impressive results. Using CompVis/stable-diffusion-v1-4 available on the HuggingFace model hub, we generate chapter artworks using the chapter description in natural language.
  • üá¨üáß English-only search: we focused on English podcasts. However, we aim to support other languages in the future üåç.
